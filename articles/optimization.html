Optimization Algorithms

// GRADIENT_DESCENT // ADAPTIVE_LEARNING // COMPUTE_EFFICIENCY

Why do we need to bother about optimization at all? Because if we are convinced that a model needs to learn, we must think about the most efficient way it can update its weights.

// FIRST_ORDER_METHODS

These are gradient-based methods that form the backbone of modern Machine Learning.

Stochastic Gradient Descent (SGD)

Pure SGD is a classic algorithm, though it is rarely used in its basic form today. "Stochastic" refers to the randomness introduced by training on a subset (mini-batch) of data.

#SGD_LOGIC 
# Key Pseudo Structure
while not converge:
    w = w - learning_rate * weight_error
    n = (n + 1) % N
return w

Momentum

Borrowed from physics, this component dampens oscillations in steep directions while accumulating velocity in shallow directions to speed up learning.

#MOMENTUM_LOGIC 
# Momentum Logic
velocity = mu * velocity - learning_rate * grad
w = w + velocity

// ADAPTIVE_ALGORITHMS

These algorithms allocate separate, automatically adapting learning rates for different parameters, giving them a significant edge over basic methods.

AdaGrad & RMSprop

AdaGrad scales learning rate based on the sum of squared gradients, but the accumulation can cause learning to stop prematurely. RMSprop addresses this by using a "leaky cache" that prioritizes recent history.

Adam & AdamW

Adam combines Momentum and RMSprop. To prevent overfitting, AdamW decouples weight decay from the gradient update—it is currently the standard for training most LLMs.

#ADAMW_LOGIC 
# AdamW Logic
m = d1 * m + (1 - d1) * grad           # Momentum
v = d2 * v + (1 - d2) * (grad**2)      # RMSprop
w -= l * m / (sqrt(v) + 1e-7)          # Update with bias correction
w -= l * rho * w                       # Decoupled Weight Decay

// LEARNING_RATE_SCHEDULING

A static learning rate is rarely optimal. We need to "switch gears" depending on the stage of learning—starting fast to escape local minima and slowing down to settle into the global optimum.

Core Concept: High LR (Exploration) → Low LR (Exploitation/Refinement)

Step Decay

Reduces the rate by a specific factor (e.g., 0.5) every few epochs. Simple, but can cause abrupt "shocks" to the gradients.

Exponential Decay

Mathematically smoothed: $\alpha = \alpha_0 e^{-kt}$. This is empirically favored for its consistent stabilization of training.

// PYTORCH_SCHEDULER_EXAMPLE 
# StepLR: Decays the learning rate of each parameter group 
# by gamma every step_size epochs.
scheduler = torch.optim.lr_scheduler.StepLR(optimizer, step_size=30, gamma=0.1)

for epoch in range(100):
    train(...)
    validate(...)
    scheduler.step() # Update the rate

1/t Decay

Reduces the rate following the factor $1/(1 + kt)$. It provides a very aggressive initial drop that mellows out over time.

// SECOND_ORDER_METHODS

These involve Gradient + Hessian calculations, such as the Newton Method or L-BFGS.

// SCALABILITY_BLOCKER

While these converge faster, they are impractical for Deep Learning. A billion-parameter model would require a billion-by-billion Hessian matrix ($O(n^2)$), making inversion computationally impossible.
